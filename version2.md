## 4. 기술 스택 (확정)

| 구분 | 기술 | 비고 |
|------|------|------|
| 프론트엔드 | React (Vite) | useState, useEffect, useRef 활용 |
| 데이터베이스 | Supabase (PostgreSQL) | 별도 백엔드 서버 없이 BaaS로 직접 연동 |
| 이미지 저장 | base64 (Supabase DB 컬럼) | 3MB 이하 이미지, 추후 Supabase Storage 전환 예정 |
| 동영상 | YouTube URL 임베드 | 직접 업로드 대신 URL 방식 사용 |
| 프론트 배포 | GitHub Pages | `npm run deploy` → gh-pages 브랜치 자동 배포 |
| 버전 관리 | Git + GitHub | main 브랜치 |

### DB 스키마 (Supabase PostgreSQL — 현재 운영 중)

```sql
CREATE TABLE public.posts (
  id          uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  created_at  timestamptz NOT NULL DEFAULT now(),
  title       text NOT NULL,
  content     text NOT NULL,
  preview     text,
  category    text NOT NULL DEFAULT '기타',
  tags        text[] NOT NULL DEFAULT '{}',
  type        text NOT NULL DEFAULT 'text'
              CHECK (type IN ('text','image','video')),
  image_data  text,      -- base64 인코딩 이미지
  video_url   text,      -- YouTube URL 등
  date        text       -- 화면 표시용 날짜 문자열 (YYYY.MM.DD)
);

-- RLS 정책 (현재: 인증 없이 모두 허용)
-- SELECT: 누구나 읽기 가능
-- INSERT: 누구나 작성 가능
-- DELETE: 누구나 삭제 가능
-- 추후 로그인 도입 시 정책 변경 예정
```

### 아키텍처 (현재)

```
[사용자 브라우저]
       │
       ▼
[React 앱 - GitHub Pages]  ← /#v2 진입 시 AppV2 렌더링
       │  Supabase JS SDK (@supabase/supabase-js)
       ▼
[Supabase Cloud - PostgreSQL]
  ─ posts 테이블
  ─ RLS (Row Level Security) 정책 적용
```
