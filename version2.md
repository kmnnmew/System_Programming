## 4. 기술 스택

| 구분 | 기술 | 비고 |
|------|------|------|
| 프론트엔드 | React | useState로 카테고리 필터·모달 상태 관리 |
| 백엔드 | Node.js + Express | 게시글 CRUD REST API |
| 데이터베이스 | MySQL | posts / categories / tags 테이블 |
| 미디어 저장 | AWS S3 또는 로컬 | 이미지·동영상 업로드 |
| 프론트 배포 | GitHub Pages | 정적 배포 |
| 백엔드 배포 | Railway 또는 Render | 무료 플랜 시작 |

### DB 설계 (예정)

```sql
-- 게시글 테이블
CREATE TABLE posts (
  id         INT AUTO_INCREMENT PRIMARY KEY,
  category   VARCHAR(30),
  title      VARCHAR(200),
  content    TEXT,
  media_url  VARCHAR(500),
  media_type ENUM('text', 'image', 'video') DEFAULT 'text',
  created_at DATETIME DEFAULT NOW()
);

-- 태그 테이블
CREATE TABLE tags (
  id      INT AUTO_INCREMENT PRIMARY KEY,
  post_id INT,
  name    VARCHAR(50),
  FOREIGN KEY (post_id) REFERENCES posts(id) ON DELETE CASCADE
);
```

### 아키텍처

```
[사용자 브라우저]
       │
       ▼
[React 앱 - GitHub Pages]  ← /#v2 진입 시 AppV2 렌더링
       │  REST API (axios)
       ▼
[Express 서버 - Railway/Render]
       │
       ▼
[MySQL DB]
```
