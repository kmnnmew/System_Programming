## 1. 프로젝트 개요

## 2. 기능 정의

### 2-1. 1단계 구현 기능 (현재)

### 2-2. 이후 단계 예정 기능 (확장 로드맵)

| 단계 | 기능 |
|------|------|

## 3. 화면 레이아웃 기획

### 3-1. 전체 페이지 구조

```
┌─────────────────────────────────┐
│           Navigation Bar        │  ← 고정 상단 바 (About, Projects, Contact)
├─────────────────────────────────┤
│         Hero Section            │  ← 이름 + 한 줄 소개 + CTA 버튼
├─────────────────────────────────┤
│         About Me                │  ← 자기소개 텍스트 + 사진(선택)
├─────────────────────────────────┤
│         Tech Stack              │  ← 기술 아이콘/태그 나열
├─────────────────────────────────┤
│         Projects                │  ← 카드형 프로젝트 목록
├─────────────────────────────────┤
│         Contact                 │  ← 이메일, GitHub, SNS 링크
├─────────────────────────────────┤
│           Footer                │  ← 저작권 표시
└─────────────────────────────────┘
```

### 3-2. 섹션별 상세 레이아웃

#### Hero Section
- 배경: 단색 또는 그라디언트
- 중앙 정렬: 이름(대형 텍스트) + 직무/소개(소형 텍스트)
- CTA 버튼: "프로젝트 보기" → Projects 섹션으로 스크롤

#### Projects Section
- 카드 그리드 레이아웃 (PC: 3열 / 모바일: 1열)
- 카드 구성: 프로젝트명, 한 줄 설명, 기술 태그, GitHub 링크

#### Contact Section
- 아이콘 + 링크 형태로 수평 나열
- 이메일 클릭 시 mailto: 링크 연결

## 5. 기술 스택 (확정)

| 구분 | 기술 | 비고 |
|------|------|------|
| 프론트엔드 | React | Vite로 프로젝트 생성 예정 |
| 백엔드 | Node.js + Express | REST API 서버 |
| 데이터베이스 | MySQL | 2단계 게시판부터 사용 |
| 프론트 배포 | Vercel | GitHub 연동 자동 배포 |
| 백엔드 배포 | Railway 또는 Render | 무료 플랜으로 시작 |
| 버전 관리 | Git + GitHub | 필수 (커밋 로그가 평가 기준) |

### 아키텍처 구조

```
[사용자 브라우저]
       │
       ▼
[React 앱 - Vercel]
       │  REST API 호출
       ▼
[Express 서버 - Railway/Render]
       │
       ▼
[MySQL DB]
```

> 1단계(기본 홈페이지)는 백엔드 없이 React 정적 페이지만 배포.  
> 2단계 게시판부터 Express + MySQL 연동 시작.

## 7. 커밋 전략 (Twin Commit 원칙)

본 프로젝트는 **Twin Commit** 방식을 따릅니다.

- 각 기능 업데이트는 **계획서(문서) + 구현** 을 한 세트로 커밋
- 문서 없이 구현만 커밋할 경우 업데이트 1개만 인정
- 문서 + 구현 동시 커밋 시 업데이트 2개 인정

```
예시 커밋 메시지:
docs: 1단계 개인 홈페이지 계획서 작성
feat: 1단계 기본 홈페이지 구현 및 배포
```
